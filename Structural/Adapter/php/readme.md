目的是客户端可以不修改旧的调用方法，就能使用新的接口实现

因为新接口的方法名、类名、实现、返回值与旧接口相比都有变化。

所以先用新实现类实例化一个新接口的实例

将新接口实例作为参数传入适配器，适配器就能将此实例转换成旧接口的实例。

就可以对转换出的旧接口实例，进行旧方法的调用了。适配器内部已经完成了调用转换。

由于新接口的返回值也会在适配器中转换成和旧接口一致的返回，所以客户端的调用和结果处理都不需要改动。只需要完成Adapter和实例定义就可以了。
